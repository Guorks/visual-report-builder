Cómo funciona el login (visual)

Este es el camino completo que recorre el operador al apretar "Conectar TikTok". Cada flecha es una transición de página o una llamada HTTP.

↑ Operador → "Conectar TikTok" → TikTok → callback en Supabase → cuenta guardada
  1. Operador hace clic en "Conectar TikTok"JWT requerido · operador autenticado en Supabase Componente ConnectTikTokButton.tsx llama tiktok-oauth-init.
  2. Frontend recibe authorize URL + state UUIDBrowser redirige a TikTok El edge function genera un par PKCE (code_verifier + code_challenge), inserta una fila en tiktok_oauth_states (vive 10 min), y arma la URL hacia TikTok con code_challenge_method=S256.
  3. El operador aprueba los permisos en TikTokSandbox = solo testers aprobados pueden completar el flujo Solo funciona si su cuenta está en la lista de 3 testers (ver abajo). Los scopes que se solicitan son user.info.basic, user.info.profile, user.info.stats, video.list.
  4. TikTok redirige al callback con un codeNo requiere JWT · TikTok llega como browser anónimo El callback (tiktok-oauth-callback) busca el state en la tabla, lo borra (single-use), y cambia el code por tokens reales usando el code_verifier.
  5. Tokens se encriptan y la cuenta se guardaSin tokens en claro en ningún lado Los tokens se cifran con AES-256-GCM + HKDF derivado por operador (master key en Supabase secrets). Se hace upsert en social_accounts con UNIQUE(platform, platform_user_id).
  6. Browser termina en /operator/social-accounts?connected=true&account_id=…Listo · operador conectado · ya puede usar "Sync Now" La página lee el query param, muestra toast de éxito, y la cuenta aparece en la lista vía realtime.

Gotchas para tener en mente

TikTok rechaza localhost. Por eso el redirect URI es la URL del edge function de Supabase, no una página del frontend. TikTok exige HTTPS y dominio público para todas las URLs de OAuth.
Sandbox y producción usan credenciales distintas. El TIKTOK_CLIENT_KEY que tenemos en Supabase ahora mismo es el de sandbox (sbawav4p7sbhufhxd1). Cuando aprueben producción, hay que swapear al client key + secret de producción. No mezclar.
El form de producción no tiene "guardar como borrador". No se puede ir llenando de a poco — TikTok rechaza el submit si falta algo. Llenar todo de una sola sentada cuando estemos listos.
El webhook authorization.removed está pendiente. Cuando un operador revoca el acceso desde TikTok, deberíamos marcarle la cuenta como revoked. Aún no está cableado (SDD §6.7.5C). Lo trabajamos después de validar el flujo principal.
